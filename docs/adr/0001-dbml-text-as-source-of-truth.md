# DBML 텍스트를 구조의 단일 진실 공급원으로 삼는다

ERD의 구조(테이블·컬럼·관계)는 DBML 텍스트로만 정의·변경하고, 사용자가 조정한 시각적 배치(테이블 위치, 관계선 경로)는 DBML과 분리된 Layout 메타데이터로 저장한다. 다이어그램에서의 시각적 구조 편집(컬럼 추가 등)을 DBML로 역생성하는 양방향 동기화는 채택하지 않는다.

## Considered Options

- **텍스트 우선 (채택)**: DBML이 진실, 배치는 별도 메타데이터. dbdiagram.io 방식. 구현이 단순하고 안정적.
- **양방향 동기화 (기각)**: 시각 편집 → 깔끔한 DBML 역생성(주석·포맷 보존 포함)은 난이도가 매우 높고, 텍스트와 그림 두 진실의 충돌 병합 문제를 떠안게 된다.
- **시각 우선 (기각)**: 사용자의 1차 요구("DBML로 정의")와 어긋난다.

## Consequences

- 다이어그램은 "읽기 + 배치 조정"만 가능하고, 구조 변경 UI는 제공하지 않는다.
- DBML 재파싱 시 기존 Layout을 보존하기 위한 안정적 식별자(테이블/엣지 키) 전략이 필요하다.
