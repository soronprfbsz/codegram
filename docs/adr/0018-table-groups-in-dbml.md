# 테이블 그룹은 DBML의 TableGroup으로 진실을 두고, 편집은 텍스트 국소 수술로 한다

> **Status**: Accepted · **Tags**: `data-model`, `rendering` · **Date**: 2026-07-15 · **Supersedes**: — · **Related**: ADR-0001, ADR-0010, ADR-0011

테이블을 색·이름 붙은 **그룹**으로 묶는 기능의 진실을 DBML의 `TableGroup` 블록에 둔다. 그룹명·소속 테이블·색은 모두 DBML 텍스트에 표현되며(ADR-0001의 "DBML=구조의 단일 진실"의 확장), 그림·색 지정·그룹 이동 UI는 이 텍스트를 소비/수술할 뿐 그림에서 그룹을 역생성하지 않는다.

사후 기록이다 — `entities/dbml/lib/groupOps.ts`(순수 텍스트 수술)·`parse.ts`(`mapTableGroup`)로 이미 구현돼 있고, 색은 `TableGroup ... [color: #hex]` 설정으로, 소속은 블록 본문의 멤버 라인으로 저장된다. 그룹별 레이아웃(ADR-0010)·그룹별 Table Doc export까지 이 진실을 따른다.

## Considered Options

### 그룹의 진실을 어디에 두나

- **DBML `TableGroup` 블록 (채택)**: 구조의 단일 진실이 DBML이라는 ADR-0001을 그룹에도 일관 적용. 그룹은 `@dbml/core`가 파싱/직렬화하는 1급 요소라 별도 저장소·동기화가 필요 없고, DB 동기화(ADR-0009)·내보내기에서 자연히 함께 흐른다.
- **Layout/별도 메타에 그룹 저장 (기각)**: 그룹을 위치정보처럼 취급하면 DBML과 그룹이 이원화돼 두 저장소 동기화 문제가 생긴다. 그룹은 위치가 아니라 **구조적 분류**라 DBML이 맞는 자리.
- **백엔드 테이블로 정규화 (기각)**: ADR-0001/0003의 "구조는 DBML 텍스트, 백엔드는 저장/스냅샷만"에 반한다.

### 그룹 편집 방식

- **RAW 텍스트 국소 수술 + 파스 가드 (채택)**: ADR-0011과 동일 정신 — 해당 `TableGroup` 영역만 재작성하고 나머지 주석·포맷·선언 순서는 보존한다. 매 연산은 결과를 재파싱해 실패 시 원본 유지(`{ ok: false }`).
- **전체 모델 재직렬화 (기각)**: 그룹 하나 바꾸는데 문서 전체를 다시 써서 사용자의 포맷·주석을 뭉갠다(ADR-0011에서 이미 기각한 방식).

### 색 저장

- **DBML `[color: #hex]` 설정 (채택)**: 색도 그룹의 속성이므로 DBML에 둔다. UI 프리셋/커스텀 색 모두 이 설정으로 수렴하고, 미지정은 설정 제거로 표현.
- **프론트 로컬/테마에 색 저장 (기각)**: 그룹과 색이 분리돼 내보내기·재로드 시 색이 유실된다.

### 식별자 인용 스타일

- **모든 그룹명·멤버를 항상 큰따옴표로 (채택)**: `@dbml/core`가 DB-sync 임포터·ModelExporter에서 모든 이름을 인용하므로, UI가 만든 `TableGroup auth { users }`가 다음 동기화에 `TableGroup "auth" { "users" }`로 뒤집히지 않게 **처음부터 all-quoted로 통일**(G1). 공백·예약어 이름은 인용이 필수라 bare로는 통일 불가.
- **미인용/조건부 인용 (기각)**: @dbml/core 정규형과 어긋나 sync마다 스타일이 표류한다.
