# 자동 배치: 희소 스키마에 dagre 계층 대신 균형 그리드 패킹을 채택한다 (그룹은 그룹별 내부 그리드 + 메타배치 — 2026-06-12 개정)

> **개정 (2026-06-12):** 그룹이 있는 스키마의 dagre-compound 유지 결정을 번복한다. dagre-LR-compound가 그룹 멤버를 FK 랭크대로 펼쳐 **그룹 박스가 세로로 길고 가운데가 비는** 문제(원 ADR가 비그룹 스키마에서 고친 "세로 붕괴"와 동일 병리)가 그룹 내부에서 그대로 재현됐다. 이제 **각 그룹 멤버를 그룹 박스 안에서 그리드로 콤팩트 패킹**하고, **그룹 블록·미분류 엔티티는 별도 메타 배치**(블록을 슈퍼노드로 보고 그룹 간 관계로 배치)로 놓는다. 이는 원 ADR가 "향후"로 기각했던 *"그룹을 (전역) 그리드에 통합"* 과 **다르다**: 그때 기각 사유였던 "전역 그리드에 멀티셀 그룹을 넣으면 배경 박스가 비멤버 셀까지 덮는다(누수)"는, 그리드를 **그룹마다 따로** 돌려 박스가 자기 멤버만 정확히 감싸므로 발생하지 않는다. 전역 Auto-arrange와 신규 **그룹별 정렬**(그룹 라벨 옆 버튼, 해당 그룹 멤버만 좌상단 기준점 유지한 채 재배치) 둘 다 이 패커를 쓴다.

DB introspect로 가져온 스키마는 테이블이 많고 FK가 드물어, dagre 계층(`rankdir: 'LR'`) 알고리즘이 고립 테이블을 전부 rank 0(맨 왼쪽 한 열)에 세로로 쌓아 "위아래로 긴" 배치를 만든다. 이를 해결하기 위해, **TableGroup이 없는 스키마는 자체 구현 그리드 패킹**으로 배치한다: 모든 테이블을 **16:10** 목표 비율의 균형 사각형 그리드에 채우되, 열 폭(240) 고정으로 열은 왼쪽 정렬·행은 상단 정렬(행 높이 = 행 내 최대 높이)되고, FK 무방향 연결성분을 BFS 순서로 이어 붙여 연결된 테이블이 인접 셀에 오게 한다(선이 짧고 간격 사이로 지남 — best-effort 비겹침). **TableGroup이 있는 스키마는 기존 dagre-compound 클러스터링을 유지**한다(균형 그리드에 멀티셀 그룹을 넣으면 그룹 배경이 비멤버 셀까지 덮는다). ADR-0003(React Flow)·ADR-0004(이름 기반 reconcile)는 유지된다.

## Considered Options

- **배치 전략**
  - **자체 그리드 패킹 (채택)**: 동기·결정적이라 reconcile이 매 렌더 동기 실행하는 현 구조를 유지한다. 열 폭 고정 덕에 행/열 정렬이 자연스럽고 16:10 균형이 나온다. 선-비겹침은 연결성 정렬 + 간격으로 best-effort.
  - ELK rectpacking (기각): `aspectRatio` 패킹 전용 알고리즘이지만 무거운 의존 + **비동기(async)** 라서 동기 reconcile 구조를 크게 바꿔야 한다.
  - 연결성분 bin-packing 하이브리드 (기각): 선 라우팅은 더 깔끔하나 블록 크기가 제각각이라 전역 그리드 정렬이 느슨해지고 구현이 복잡하다.
- **선-엔티티 비겹침**
  - **연결성 정렬 + 간격 (채택)**: 단순·동기. 밀집 클러스터엔 일부 교차가 남을 수 있고, 이는 드래그 정렬 스냅으로 보정한다.
  - 장애물 회피 라우팅 (기각): 비겹침을 보장하지만 라우팅 알고리즘 + 엣지 렌더링 복잡도가 크다.
- **그룹 처리**
  - ~~조건 분기 — 그룹 없으면 그리드, 있으면 dagre 유지 (채택)~~ **(2026-06-12 번복)**: 그룹 클러스터링은 보존됐으나, dagre-LR이 그룹 내부에서 멤버를 펼쳐 박스가 세로로 길고 비는 문제가 재현됐다.
  - **그룹별 내부 그리드 + 메타배치 (2026-06-12 채택)**: 각 그룹 멤버를 그룹 박스 안에서 그리드 패킹하고, 그룹 블록·미분류는 메타 배치로 놓는다. 그리드를 그룹마다 따로 돌리므로 박스가 자기 멤버만 감싸 배경 누수가 없다.
  - 그룹을 (전역) 그리드에 통합 (기각, 유지): 하나의 전역 균형 그리드에 멀티셀 그룹 블록을 넣으면 배경 박스가 비멤버 셀까지 덮는 누수·정렬 손실이 생긴다. (위 "그룹별 내부 그리드"는 전역이 아니라 그룹마다 따로 돌리는 점에서 다르다.)

## Consequences

- `autoLayout`은 그룹 없음 → 단일 그리드, 그룹 있음 → 그룹별 내부 그리드 + 블록 메타배치(2026-06-12). 둘 다 그리드 기반이라 그룹 추가/제거 시 배치 "스타일"의 급변은 줄었다.
- **그룹별 정렬(부분 재배치)은 국소적이다**: 해당 그룹 멤버만 좌상단 기준점을 유지한 채 재배치하고 다른 그룹·테이블은 건드리지 않으며, 이동한 멤버에 닿는 엣지의 수동 경로만 초기화한다(전역 Auto-arrange가 모든 수동 경로를 제거하는 것과 대비 — CONTEXT.md "수동 경로" 참조).
- 그리드는 동기·순수·결정적이라 reconcile(ADR-0004) 베이스라인·Auto-arrange·신규 프로젝트에 그대로 적용된다. 저장된 수동 위치는 reconcile이 우선 보존한다.
- 선-비겹침은 best-effort(연결성 정렬 + 간격)다. 밀집 클러스터의 잔여 교차는 **드래그 정렬 가이드·스냅**(별도 하위 기능)으로 사용자가 보정한다.
- 드래그 정렬 가이드·스냅은 자동 배치와 독립이며, 이동 시 다른 노드와 좌/중/우(x)·상/중/하(y) 6축으로 정렬을 감지해 가이드선 표시 + 스냅한다.
