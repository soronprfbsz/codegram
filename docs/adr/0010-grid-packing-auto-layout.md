# 자동 배치: 희소 스키마에 dagre 계층 대신 균형 그리드 패킹을 채택한다 (그룹 시 dagre 유지)

DB introspect로 가져온 스키마는 테이블이 많고 FK가 드물어, dagre 계층(`rankdir: 'LR'`) 알고리즘이 고립 테이블을 전부 rank 0(맨 왼쪽 한 열)에 세로로 쌓아 "위아래로 긴" 배치를 만든다. 이를 해결하기 위해, **TableGroup이 없는 스키마는 자체 구현 그리드 패킹**으로 배치한다: 모든 테이블을 **16:10** 목표 비율의 균형 사각형 그리드에 채우되, 열 폭(240) 고정으로 열은 왼쪽 정렬·행은 상단 정렬(행 높이 = 행 내 최대 높이)되고, FK 무방향 연결성분을 BFS 순서로 이어 붙여 연결된 테이블이 인접 셀에 오게 한다(선이 짧고 간격 사이로 지남 — best-effort 비겹침). **TableGroup이 있는 스키마는 기존 dagre-compound 클러스터링을 유지**한다(균형 그리드에 멀티셀 그룹을 넣으면 그룹 배경이 비멤버 셀까지 덮는다). ADR-0003(React Flow)·ADR-0004(이름 기반 reconcile)는 유지된다.

## Considered Options

- **배치 전략**
  - **자체 그리드 패킹 (채택)**: 동기·결정적이라 reconcile이 매 렌더 동기 실행하는 현 구조를 유지한다. 열 폭 고정 덕에 행/열 정렬이 자연스럽고 16:10 균형이 나온다. 선-비겹침은 연결성 정렬 + 간격으로 best-effort.
  - ELK rectpacking (기각): `aspectRatio` 패킹 전용 알고리즘이지만 무거운 의존 + **비동기(async)** 라서 동기 reconcile 구조를 크게 바꿔야 한다.
  - 연결성분 bin-packing 하이브리드 (기각): 선 라우팅은 더 깔끔하나 블록 크기가 제각각이라 전역 그리드 정렬이 느슨해지고 구현이 복잡하다.
- **선-엔티티 비겹침**
  - **연결성 정렬 + 간격 (채택)**: 단순·동기. 밀집 클러스터엔 일부 교차가 남을 수 있고, 이는 드래그 정렬 스냅으로 보정한다.
  - 장애물 회피 라우팅 (기각): 비겹침을 보장하지만 라우팅 알고리즘 + 엣지 렌더링 복잡도가 크다.
- **그룹 처리**
  - **조건 분기 — 그룹 없으면 그리드, 있으면 dagre 유지 (채택)**: 그룹 클러스터링 보존과 그리드 정렬을 둘 다 확보한다. 두 경로지만 각 경우에 최적.
  - 그룹을 그리드에 통합 (기각, 향후): 균형 그리드에 멀티셀 그룹 블록을 넣으면 배경 박스 누수·정렬 손실이 생긴다.

## Consequences

- `autoLayout`이 그룹 유무로 분기한다(그룹 없음 → 그리드, 있음 → dagre). 사용자가 그룹을 추가/제거하면 자동 배치 스타일이 바뀔 수 있다.
- 그리드는 동기·순수·결정적이라 reconcile(ADR-0004) 베이스라인·Auto-arrange·신규 프로젝트에 그대로 적용된다. 저장된 수동 위치는 reconcile이 우선 보존한다.
- 선-비겹침은 best-effort(연결성 정렬 + 간격)다. 밀집 클러스터의 잔여 교차는 **드래그 정렬 가이드·스냅**(별도 하위 기능)으로 사용자가 보정한다.
- 드래그 정렬 가이드·스냅은 자동 배치와 독립이며, 이동 시 다른 노드와 좌/중/우(x)·상/중/하(y) 6축으로 정렬을 감지해 가이드선 표시 + 스냅한다.
