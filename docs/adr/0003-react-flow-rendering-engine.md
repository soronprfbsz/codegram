# ERD 캔버스 렌더링 엔진으로 React Flow(@xyflow/react) v12를 사용한다

ERD 캔버스(테이블 노드, 관계선 엣지, 드래그·줌·팬, 테이블그룹 영역)는 React Flow v12로 구현한다. 테이블은 커스텀 노드로, 컬럼마다 핸들을 두어 관계선을 특정 컬럼에 정확히 연결한다. 관계선은 커스텀 엣지로 까마귀발 마커를 그린다. 테이블그룹 영역은 부모(그룹) 노드 또는 배경 레이어로 표현한다.

## Considered Options

- **React Flow v12 (채택)**: MIT, 컬럼별 핸들/커스텀 엣지/노드 그룹 지원, ERD에 가장 적합, 생태계·문서 풍부.
- **커스텀 SVG + d3-zoom (기각)**: 드래그·줌·라우팅을 전부 직접 구현 → 막대한 비용.
- **GoJS/JointJS+ (기각)**: 강력하나 상용 라이선스 비용·lock-in.
- **maxGraph (기각)**: 무겁고 React 통합 수작업 필요.

## Consequences

- 캔버스 코드가 React Flow API에 결합된다(교체 시 캔버스 전면 재작성). 공식·표준 라이브러리이므로 수용.
- Layout(위치·엣지 경로)은 React Flow의 노드/엣지 좌표 모델로 직렬화해 저장한다.
