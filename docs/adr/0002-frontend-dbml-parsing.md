# DBML 파싱은 프론트엔드에서 공식 @dbml/core로 수행한다

DBML 텍스트를 구조화 모델로 변환하는 파싱·검증·렌더링은 React 프론트엔드에서 공식 `@dbml/core`(JS, Holistics 관리)로 수행한다. FastAPI 백엔드는 DBML 원본 텍스트·Layout·프로젝트·인증을 저장하는 얇은 영속화 계층으로 둔다. 서버 측 산출물이 필요하면 프론트가 이미 파싱한 JSON 모델을 백엔드로 전달한다.

## Considered Options

- **프론트 단독 (채택)**: 공식 파서로 스펙 100% 커버리지, 즉각적 미리보기, 백엔드 단순화.
- **백엔드 단독 (기각)**: Python 파서(pydbml v1.2.1 / lark-dbml)는 커뮤니티 관리로 캐논 DBML 스펙을 후행 → 최신 문법 호환·버그 리스크.
- **Node 사이드카 (기각)**: 공식 파서를 서버에서 쓸 수 있으나 Python+Node 혼합 운영 부담.
- **하이브리드 (기각)**: 두 파서 결과 불일치와 이중 유지보수 비용.

## Consequences

- FastAPI는 DBML 의미를 "이해"하지 않는다. 서버 생성 정의서/PDF 등은 프론트가 보낸 파싱 모델에 의존한다.
- 파서 lock-in은 `@dbml/core`에 집중된다(공식이므로 수용 가능한 리스크).
