# 시각 스타일은 디자인 토큰으로만 표현하고 raw 하드코딩을 금지한다 (타이포 스케일 토큰화 포함)

> **Status**: Accepted · **Tags**: `frontend`, `architecture` · **Date**: 2026-07-15 · **Supersedes**: — · **Related**: ADR-0007, ADR-0017

프론트엔드의 모든 시각 스타일(색·폰트 크기)을 **디자인 토큰으로만** 표현한다. raw 값(Tailwind 팔레트 클래스 `text-red-600`, 인라인 `#hex`/`rgb()`, 인라인 `fontSize: 13`, `text-[13px]`)을 호출부에 박는 것을 금지한다. 기존 토큰으로 표현 불가하면 **먼저 토큰을 신규 추가**한 뒤 그 토큰으로 구현한다. 이는 F1/F2(공용 토큰·컴포넌트)와 ADR-0007(shadcn 토큰 시스템)의 원칙을 색·크기 전반에 강제하는 확장이며, 이 프로젝트엔 폰트 크기 토큰이 없어 **타이포 스케일을 신규 토큰화**한 결정을 함께 기록한다.

배경: 각 페이지가 팔레트 색(`text-gray-600`·`text-red-600`)과 인라인 `fontSize` 숫자(60여 곳, 8.5~22px 근사 중복)로 스타일을 재구현해, 같은 역할 UI가 화면마다 표류하고 테마·브랜드 일괄 변경이 불가능했다. 규칙 문서(`.claude/rules/frontend.md` F5)와 훅(`inject_rules.py` 주입 + `lint_rules.py` F5 lint)으로 강제한다.

## Considered Options

### 시각 스타일 값의 출처

- **토큰 온리 + 없으면 토큰 추가 (채택)**: 색은 shadcn 시맨틱 토큰(`text-muted-foreground`·`-destructive`·`-success`·`-warning`)·`--erd-*`, 폰트는 named step·`--erd-fs-*`. 부족하면 `src/index.css`(라이트/다크)에 토큰을 추가한 뒤 사용. 단일 출처에서 테마·브랜드를 일괄 제어.
- **raw 값 허용(관례로만 권장) (기각)**: 강제가 없으면 페이지마다 재유입돼 표류. 실제로 그 상태였다.
- **CSS-in-JS 테마 객체 도입 (기각)**: 새 의존성·런타임 비용. 기존 Tailwind 토큰 + CSS 변수로 충분(ADR-0007).

### 폰트 크기 스케일을 어디에 두나

- **일반=Tailwind named step 유지 + ERD=`--erd-fs-*` 신규 (채택)**: `text-sm`이 87곳에서 14px로 쓰여, Tailwind 기본 스텝을 재정의하면 앱 전체가 축소된다. 그래서 일반 UI는 표준 named step(그 자체가 토큰)을 유지하고 작은 라벨용 `--text-2xs`(10px)만 추가. ERD 캔버스는 픽셀 튜닝(노드 크기·라우팅이 텍스트 메트릭에 의존)이라 별도 `--erd-fs-*`(10·11·12·13·14·15·22)로 현재 px를 보존하며 진짜 중복만 수렴(8.5/9/10.5→10, 12.5→12). 색을 F2에서 "일반=shadcn, ERD=`--erd-*`"로 나눈 것과 동일한 분리.
- **Tailwind `--text-*` 전면 재정의로 단일 스케일 (기각)**: 이름있는 클래스 하나로 통일되나, `text-sm`(87곳)·`text-xs`(30곳)의 픽셀 의미가 바뀌어 앱 전체가 예기치 않게 축소되고, ERD의 픽셀 튜닝을 강제 스냅해 레이아웃 회귀 위험이 크다.
- **모든 기존 px를 별도 값 토큰으로 1:1 보존 (기각)**: 표류 제거(근사 중복 수렴)라는 목적을 잃고 토큰만 난립.

### 강제 방식

- **규칙(F5) + 주입 훅 + PostToolUse lint (채택)**: `.tsx` 편집 후 raw 팔레트 클래스·인라인 hex·`fontSize:숫자`·`text-[NNpx]`를 advisory로 flag(비차단, 기존 훅 철학과 일관). 도메인 데이터(글리프·프리셋)·설정(Monaco 테마)은 예외로 명시.
- **차단(blocking) lint (기각)**: 정당한 예외(데이터 hex)를 막아 작업이 끊긴다. advisory로 충분.
