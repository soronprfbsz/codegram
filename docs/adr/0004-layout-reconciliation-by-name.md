# Layout은 DBML의 이름 기반 키로 보존하고, 이름 변경 시 위치 손실을 허용한다

DBML에는 안정적 ID가 없으므로, Layout은 테이블을 `(스키마.)이름`으로, 엣지를 참조 컬럼 시그니처로 식별한다. DBML 파싱이 성공할 때마다 저장된 Layout과 머지한다: 이름이 유지된 항목은 위치/웨이포인트를 유지하고, 신규 항목은 자동 배치하며, 사라진 항목은 정리한다. 테이블/컬럼 이름을 바꾸면 새 항목으로 취급되어 기존 수동 위치는 손실된다.

## Considered Options

- **이름 기반 키 + 머지 (채택)**: 단순·예측 가능. 이름변경 시 위치 손실은 수용.
- **이름변경 휴리스틱 매칭 (기각)**: 위치는 더 잘 보존되나 오탐 위험과 구현 복잡도.
- **DBML에 안정 ID 주입 (기각)**: 위치는 완벽 보존되나 사용자의 DBML을 오염시켜 이식성·가독성을 해침(ADR-0001 정신 위배).

## Consequences

- 사용자가 대량 리네이밍을 하면 수동 배치가 사라질 수 있다 → UX 상 "이름 변경은 위치를 잃을 수 있음"을 알리거나, 향후 휴리스틱 매칭을 옵션으로 추가 가능.
- 신규/미배치 테이블을 위한 자동 배치(auto-layout)가 반드시 필요하다.
