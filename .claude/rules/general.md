# General rules — Codegram (모든 코드 작업 공통)

프론트/백엔드를 가리지 않고 **모든** Codegram 코드 작업에 적용되는 공통 규칙. 영역별 규칙(`frontend.md`/`backend.md`)과 함께 읽는다. 도메인 언어는 `CONTEXT.md`, 아키텍처 결정은 `docs/adr/`.

## G1. 단일 출처 (중복 금지)
- 같은 로직·모양·값은 **한 곳에서 정의하고 재사용**한다. 호출부마다 복붙·재구현하지 않는다(프론트는 디자인 토큰/컴포넌트, 백엔드는 repository/service/util, 문서는 이 rules 체계 자체).
- 리뷰/리팩터에서 "같은 것을 또 구현"한 코드는 결함으로 보고 공용 단위로 회수한다.

## G2. 변경은 외과적으로
- 요청에 직접 연결되는 라인만 바꾼다. 인접 코드·포맷·주석을 "개선"하지 않고, 기존 스타일·관례를 따른다.
- 내 변경이 만든 고아(미사용 import/상수/함수)는 정리한다. 기존 dead code는 **언급만** 하고 지우지 않는다.

## G3. 종료 전 검증 (주장 아니라 증거)
- 완료를 주장하기 전에 검증 명령을 실제로 돌려 **출력으로 증명**한다(영역별 명령은 `frontend.md`/`backend.md`).
- 실패·생략·미실행은 그대로 보고한다. 성공만 골라 말하지 않는다.
- 추측하지 말고 측정한다: 재현·프로브로 원인을 확정한 뒤 고친다.

## G4. 사전 이슈 vs 회귀 구분
- 테스트/동작 실패가 내 변경 때문인지 원래 있던 것인지 구분한다. 불확실하면 `git stash`로 main 상태와 대조해 확인한다.
- 사전 존재(내 변경과 무관) 실패는 그렇게 **명시**하고, 범위 밖이면 임의로 건드리지 않는다.

## G5. 조심해야 할 작업
- 되돌리기 어렵거나 외부로 나가는 작업(커밋·푸시·배포·외부 전송·삭제/덮어쓰기)은 사전 승인 없이는 먼저 확인한다.
- 삭제·덮어쓰기 전에 대상을 직접 확인한다 — 설명과 다르거나 내가 만들지 않은 것이면 진행 대신 그 사실을 알린다.

## G6. Design-first — 아키텍처 결정은 ADR 먼저
코드 작업 시작 전, 이 요청이 **아키텍처 결정을 신설/변경하는지** 판단한다(아래 기준). 유의미하면 **구현 전에** ADR을 확정하고 사용자 승인을 받는다(design-first). 아니면 그대로 진행하되, 경계선상이면 "ADR 불요" 한 줄로 판단을 남긴다.

- **ADR 대상(구현 전 ADR 확정):** 의존성/외부 시스템 추가·제거 · source-of-truth/데이터 흐름 계약 변경(예: DBML=진실 ADR-0001) · 횡단 메커니즘 도입(인증·캐시·스케줄·버전관리) · 계층/서비스 경계 계약 변경 · 되돌리기 비싸고 "왜 이렇게?"가 남을 결정.
- **ADR 비대상(그냥 진행):** 버그 픽스 · 스타일/카피/i18n 문구 · 계층 내부 리팩터 · 테스트 추가 · 설정 조정.
- **탐색·판단:** 작성 전 `docs/adr/README.md` 인덱스로 **해당 도메인의 기존 ADR을 통독** → amend(supersede)/extend/none 판단. 결정 **변경은 편집이 아니라 새 ADR로 supersession**(옛 것에 `Superseded-by`, 새 것에 `Supersedes`). 도메인은 파일 계층이 아니라 **태그·인덱스**로 묶는다(번호는 불투명 단조 ID).
- **작성:** 신규는 순번 `docs/adr/NNNN-kebab-title.md` + 헤더 한 줄(`Status`/`Tags`/`Date`/`Supersedes`/`Related`) + `## Considered Options`(채택/기각) + **인덱스 행 추가**. 도메인 언어(`CONTEXT.md`)를 바꾸면 그 사실도 ADR에 남긴다(ADR-0015·0016 선례). 초안은 `grill-with-docs` 스킬로 도메인 모델에 대질하며 다듬는다.
- **훅:** `UserPromptSubmit`(`adr_gate.py`)가 세션 첫 요청에 이 판단을 상기시키고, 고신호 경로(의존성 매니페스트·`models/`) 편집 시 `lint_rules.py`가 ADR 확인을 advisory로 띄운다(둘 다 비차단).
